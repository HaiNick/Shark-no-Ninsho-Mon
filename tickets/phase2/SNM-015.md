## Problem

Current regex allows edge cases. Missing RFC 5321 length checks.

## Fix

Add max length check (`len(email) > 254`), consecutive dot check (`..`), and split-and-validate local/domain parts independently.

## PR Checkpoints

- [ ] Emails longer than 254 characters rejected
- [ ] Local part longer than 64 characters rejected
- [ ] Consecutive dots (`..`) rejected
- [ ] Existing valid email addresses still accepted
- [ ] Unit tests cover boundary cases
