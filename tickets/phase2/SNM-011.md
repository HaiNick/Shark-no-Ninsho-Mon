## Problem

Health checks run sequentially with no timeout. One slow backend blocks all checks.

## Fix

Use `ThreadPoolExecutor(max_workers=5)` with 10s per-check timeout. Implement exponential backoff: `min(60 * 2^failures, 1800)` seconds between retries for failing routes.

## PR Checkpoints

- [x] `ThreadPoolExecutor` used with `max_workers=5`
- [x] Per-check timeout of 10 seconds enforced
- [x] Exponential backoff implemented (max 30 min)
- [x] Failure counts reset to 0 on successful check
- [x] Health check logs include how many routes were checked per cycle
