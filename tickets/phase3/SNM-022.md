## Problem

`fix_file_permissions()` is overly complex, follows symlinks, and crashes on permission errors instead of warning.

## Fix

Simplify `fix_file_permissions()`. Skip symlinks. Catch `OSError`/`PermissionError` separately. Add fallback warning if `SUDO_UID`/`SUDO_GID` can't be parsed.

## PR Checkpoints

- [ ] Symlinks are skipped during permission fixing
- [ ] `OSError` and `PermissionError` caught and logged as warnings (not crashes)
- [ ] Warning shown if `SUDO_UID`/`SUDO_GID` cannot be parsed
- [ ] Function behaves correctly when not running as root
