# SNM-020 · LOW

**Fix client-side cache expiration in admin.js**

| Field | Value |
|-------|-------|
| **Priority** | LOW |
| **Phase** | 3 — Long Term (Next Quarter) |
| **File** | `app/static/js/admin.js:39-46` |

---

## Problem

Raw `localStorage` usage without expiration — stale data persists indefinitely.

## Fix

Replace raw `localStorage` with a `CacheManager` wrapper that stores `{ data, timestamp }` and respects a configurable `maxAge` (default 300s). Add `invalidate(pattern)` for cache busting after mutations.

## PR Checkpoints

- [ ] `CacheManager` wrapper implemented in `utils.js`
- [ ] Stale entries (older than `maxAge`) automatically evicted on read
- [ ] `invalidate(pattern)` called after all route create/update/delete operations
- [ ] No regressions in dashboard data loading behaviour
