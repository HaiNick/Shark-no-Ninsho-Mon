# SNM-018 · LOW

**Standardize error handling with custom exception classes**

| Field | Value |
|-------|-------|
| **Priority** | LOW |
| **Phase** | 3 — Long Term (Next Quarter) |
| **File** | `app/errors.py` (new file) |
| **Status** | ✅ Done |

---

## Problem

Inconsistent error responses across endpoints. No custom exception hierarchy.

## Fix

Create `AppError`, `ValidationError`, `AuthorizationError`, `NotFoundError` with status codes. Register `@app.errorhandler(AppError)` to return consistent JSON error responses.

## PR Checkpoints

- [x] `app/errors.py` created with all four exception classes
- [x] `@app.errorhandler(AppError)` registered in `app.py`
- [x] All endpoints return consistent `{"error": "..."}` JSON on failure
- [x] HTTP status codes correct for each error type (400, 403, 404, 500)
