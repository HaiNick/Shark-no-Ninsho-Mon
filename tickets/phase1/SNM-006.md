## Problem

Concurrent reads/writes to `emails.txt` are not atomic — file can corrupt under simultaneous admin actions.

## Fix

```python
import fcntl, contextlib

@contextlib.contextmanager
def locked_file(path, mode='r'):
    with open(path, mode, encoding='utf-8') as f:
        fcntl.flock(f.fileno(), fcntl.LOCK_EX)
        try:
            yield f
        finally:
            fcntl.flock(f.fileno(), fcntl.LOCK_UN)
```

Use `locked_file` for all email read/write operations.
