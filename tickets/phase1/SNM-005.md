# SNM-005 · HIGH

**Add CSRF protection to all state-changing endpoints**

| Field | Value |
|-------|-------|
| **Priority** | HIGH |
| **Phase** | 1 — Immediate (Before Production) |
| **File** | `app/app.py` (all POST/PUT/DELETE routes) |
| **Status** | ✅ Done |

---

## Problem

No CSRF tokens. An authenticated user can be tricked into triggering route deletions, email list changes, etc.

## Fix

```python
from flask_wtf.csrf import CSRFProtect, generate_csrf
csrf = CSRFProtect(app)

@app.after_request
def inject_csrf(response):
    response.headers['X-CSRF-Token'] = generate_csrf()
    return response
```

```html
<!-- base.html -->
<meta name="csrf-token" content="{{ csrf_token() }}">
```

```javascript
// utils.js — add to all fetch/XHR calls
headers['X-CSRF-Token'] = document.querySelector('meta[name="csrf-token"]')?.content;
```

Add `flask-wtf>=1.2.1` to `requirements.txt`.
