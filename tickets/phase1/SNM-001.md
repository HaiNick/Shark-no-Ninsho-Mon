# SNM-001 · HIGH

**Remove Caddy Admin API public port exposure**

| Field | Value |
|-------|-------|
| **Priority** | HIGH |
| **Phase** | 1 — Immediate (Before Production) |
| **File** | `docker-compose.yml:96` |
| **Status** | ⬜ Open |

---

## Problem

Port 2019 is publicly mapped — anyone on the network can modify Caddy routing without authentication.

## Fix

```yaml
# Remove this line from caddy service:
# - "2019:2019"
```

Flask still reaches Caddy at `http://caddy:2019` via the internal Docker network. No app code changes needed.

## PR Checkpoints

- [ ] Port `2019:2019` mapping removed from `docker-compose.yml`
- [ ] Caddy Admin API no longer reachable from host (`curl http://localhost:2019` fails)
- [ ] Flask can still sync routes to Caddy via internal network (smoke test route add/delete)
- [ ] `docker compose up` starts without errors
